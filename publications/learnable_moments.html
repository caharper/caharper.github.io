                                        TL;DR

                                        Statistical moment orders are not differentiable, making them difficult to learn in traditional neural networks. We propose a continuous relaxation, a smooth ramp function, as a proxy to learn the statistical moment order. Our methods outperform traditional fixed moment pooling layers in automatic modulation classification tasks.

                                        # Paper Contributions

                                        1. To the best of our knowledge, the first differentiable method of learning statistical moment order for pooling layers in neural networks.
                                          1. This reduces the need for hyperparameter tuning and allows the network to learn the most useful moments for the task at hand directly from the input data.
                                        2. Improved performance in automatic modulation classification tasks over traditional fixed moment pooling layers.

                                        # Paper Purpose

                                        The purpose of this work is to develop a learnable pooling layer that can be used to extract statistical moments from input signals. Most of the time when we use statistical moments, we use a fixed set of moments, usually mean and variance/standard deviation. Higher-order moments are also used in some cases, skewness and kurtosis for example. Taking statistical aggregations has a few primary purposes:

                                        1. Reducing the dimensionality of the input signal. This reduces network complexity, can reduce overfitting, and can speed up training and inference.
                                        2. Introducing an information bottleneck, forcing the network to learn the most important features.
                                        3. Reducing the sensitivity of the network to noise ("aggregating out" the noise).
                                        4. Enables the processing of variably-shaped input signals. For example, no matter the length of the input signal, the mean will still be a scalar. This is important in signal processing (e.g., speech recognition and time series analysis) as signals are often of variable durations/length.

                                        The problem is that we don't know which moments are the most useful for a given task. Using fixed moments, we are stuck defining the moments as a hyperparameter, and we have to hope that the moments we choose are the most useful. To get to the "best" solution, we have to try many different combinations of moments, which is computationally expensive and time-consuming. Instead, we'd like to allow the network to directly learn which moments are most useful for the task at hand. But this isn't as easy as it sounds.

                                        Neural networks, in their current form, rely on gradient descent to optimize network weights. This means that the network must be differentiable, and the pooling layer must be differentiable. This is a problem because the moment order is not differentiable. Before getting to that, let's have a look at a brief background on statistical moments.

                                        # What is a Statistical Moment?

                                        A statistical moment is a measure of the shape, location, or spread of a distribution, xâˆˆRN, where N is the number of samples. The most common that you've probably heard of are the mean and variance. The mean is a measure of the location of the distribution, and the variance is a measure of the spread. However, there are many other moments that can be used to describe a distribution. These three measures can be broken into three categories of statistical moments: raw, central, and standardized moments. Let's dive into each one.

                                        # The Problem with Learning Moments

                                        Now that we know what statistical moments are, let's talk about the problem with learning them. Ideally, we would allow the network to just learn the statistical moment order, k. In this way, the network could learn which moments are most useful for the task at hand, whether it be skewness or kurtosis when using standardized moments for example. However, the moment order is not differentiable. This means that we can't just plug the moment order into the network and expect it to learn the best moment order. It won't work with gradient descent! Let's see why.

                                        So how do we solve these problems while still learning an actual statistical moment? Let's look at our solution next.

                                        # Making Moments Learnable

                                        We've already seen how to make the raw moments learnable by using x^=ReLU(x)+Ïµ. Now, we need to make central and standardized moments learnable.

                                        Learned moments would be ideal, but they have gradient issues. Fixed moments don't have gradient issues, but they are not learnable. The question is how do we make moments learnable while avoiding gradient issues? Can we somehow "trick" the network into thinking it is using a fixed moment, but it is actually learning the moment? In our paper, we make this answer yes!

                                        We use a smooth ramp function as a proxy to learn the moment order, k. This allows us to learn the moment order, k, directly from the input data, and it allows us to use gradient descent to optimize the moment order, k. This is a big deal! This means that we can learn the most useful moments for the task at hand, and we don't have to rely on a fixed set of moments as hyperparameters. Let's take a look at that ramp function below.

                                        Smooth Ramp Function

                                        The idea of this function is to learn the position of the ramp decaying from 1 to 0 in R steps. R is a hyperparameter that we can set so that that the ramp is smooth. If R=0, we have a step function. If Râ‰¥1 and RâˆˆZ, we have a smooth ramp. This makes the function differentiable meaning that we can use gradient descent to optimize the location of the ramp.

                                        Now that we have this function, by utilizing it in the correct way, we can make a proxy for optimizing k.

                                        Our Solution:

                                        Instead of applying k directly, we "trick" the network into thinking it is using a fixed order, k^. Since fixed moments do not have a gradient with respect to k, we don't get the gradient instability issues that we had before. We accomplish this "trick" by using the stop gradient operator[2]. This allows us to use non-differentiable operations, like rounding, in the network. To make sure the gradient still flows, we provide an alternative path so that we can optimize via gradient descent. This is our proxy for learning the moment order, k.

                                        Although k^ allows us to learn the statisical moment order by avoiding gradient instability, it still needs careful consideration. From the previous section, we know that if k^âˆˆR, we will get complex-valued activations from the layer where xj<Î¼x. This is a problem because most neural network layers and loss functions assume that the gradient and activations are real-valued. If k^âˆˆZ, (i.e., k^ is an integer), then the (â‹…)k^ activations will be real-valued for all xjâˆˆx. To accomplish this, within the stop gradient operator, we run the following:

                                        int_val = tf.stop_gradient(tf.where(tf.cast(ramp_vals, tf.float32) > 0.0)[:, 0])

                                        This finds the index of the non-zero values of the ramp function, and we use the maximum index as the learned moment order, k^. This ensures that activations will be real-valued and defined for all xjâˆˆx.

                                        Important

                                        Because moment orders are always non-negative, we ensure that k>0 (if k=0 then the moment is just a function of the number of samples, N). To achieve this, we clip the learned value of k to be in the following ranges for each moment:

                                        • Raw moments: kâˆˆ(0,6]
                                          • The upper bound of 6 is chosen to avoid overflow.
                                        • Central moments: kâˆˆ[2,6]
                                          • The upper bound of 6 is chosen to avoid overflow.
                                          • The lower bound of 2 is chosen to avoid a constant value regardless of the input as discussed in Central Moments.
                                        • Standardized moments: kâˆˆ[3,6]
                                          • The upper bound of 6 is chosen to avoid overflow.
                                          • The lower bound of 3 is chosen to avoid a constant value regardless of the input as discussed in Standardized Moments.

We show that our method outperforms traditional fixed moment pooling layers in automatic modulation classification tasks. We also show that our method is more robust to noise and has better generalization capabilities. We hope that our method will be useful for other tasks as well.
